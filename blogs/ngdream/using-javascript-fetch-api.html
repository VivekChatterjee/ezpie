Javascript fetch API

In this article you will get to know about the Javascript's fetch API

What is fetch API?

The Fetch API is the modern way of making HTTP requests to servers from a web browsers. The Fetch API is quite simpler and understandable, plus you may have used the XMLHttpRequest, which is al lot longer and less understandable than the fetch API. More on, the fetch API can do things the same way the XMLHttpRequest does.

How to read the responce?

If the data is in text form you can use the text() method. This method returns the data in a simple text format.

But the best practice is to use the async and await, we use this becuase this provides us an alternative syntax for handling the responce of the Fetch request and other methods.

For this time we are using the text() method, but this is not possible for json, form data, etc. For them the fetch() method have json(), formData(),arrayBuffer() and more.

Handling status code

This is one of the most known responce you may have seen on typing a wrong url on a site:
404

But have you ever though what this is? Well if no then here's your answer

This is called a status code, what this means is that, while you send a request to the server of a site there was some error on your side, but there was no error on the server side, so basically this means that you typed the url wrong.

In requesting APIs from the server you should always tell the status code, because it helps us to do actions as per the situation.

Say, you got a status code of 200/OK, that would mean that you should display a message telling the client that their action was successful and if it gives a status code of 404/page not found then the other way round.
