What is GitHub? How to use it

In this article you will get to know about what is github and why it's so much used.

GitHub is not like Git

GitHub is not like Git in many ways, but here are the most common reasons why GitHub is not even close to Git:

Difference Git GitHub
Environments Git is installed locally on our system, so that we can manage our source code using our computer as a repository Github, on the other hand, works in the internet(cloud), so Internet access is important. It also has a built-in user management system and a user-friendly Graphical User Interface(GUI). More all git can be used without GitHub, but GitHub can't be used without Git.
Functionality Git is a free, open-source software Distributed Version Control System (DVCS) created to manage all source code history. It can keep a history of commits, it can reverse changes, and let us share code using GitHub. GitHub, on the other hand, is a hosting service for Git repositories. It offers all of the DVCS and SCM Git can offer, but with some additional features. Which are: project management, bug tracking etc.
Owners are diffrent Git was developed by linux founder Linus Torvalds and is also maintened by the linux foundation. GitHub, on the other side, was created as a totally diffrent company, but was acquired by Microsoft later on.

From this you can make out that GitHub and Git are totally diffrent from each other.

What is GitHub

Now we now that Git and GitHub are diffrent, lets understand what GitHub is.

GitHub is a service that allows developers to store, track, and collaborate on a project of their own or others. GitHub serves as a social networking site in which developers can openly network, collaborate, and pitch their work with other developers.

With this, GitHub also encourages users to create their personal profile. You can visit anyone's profile and see what projects they own and contribute to. This some how makes GitHub a type of social network for programmers 🤣

What is GitHub used for?

GitHub allows developers to create remote, public-facing repositories on the cloud for free. Once you have set up a repository on GitHub, you can copy it to your computer, and then modify files locally, then you can commit your changes back to the repository at GitHub, where your changes are displayed to the public.

How to setup your first GitHub profile

To setup GitHub, you must make sure you have some idea of what git is and have it installed in your machine.

Here's how you can setup GitHub in the simplest way:

First you need to create an account in GitHub

Now, lets make a repo to get started! Click on the plus sign in the top right corner to add your repository

Next, we are going to fill up the details of our repository.

For this I have already setup a local git repository, if you have not then you can just use any of your old git repository. For this we just need to copy the url given

Now paste the url after typing this command

> git remote add origin https://github.com/ishaan010/Hello-github.git

git remote add origin tells git that you are adding a remote repository, with a specified URL, as an origin to your local Git repository.

Now we are going to push our local code to the GitHub repository so that it's visiable to everone

> git push --set-upstream origin master

NOTE: This is not how you will push your code ever time, you will do it like this: git push origin, but you will do this after you have pushed your first commit on the GitHub repository with git push --set-upstream origin master

Now go to your GitHub repository and see what it's showing.
